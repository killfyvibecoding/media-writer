# FAQ

## Media Writer 是命令行工具吗？

它首先是一个 Codex / Agent Skill，不是独立完整 CLI。仓库中的脚本用于让 Agent 在关键步骤上稳定执行，例如生成封面、构建草稿 JSON、转换微信 HTML。

## 必须使用 NotebookLM 吗？

播客、NotebookLM PPT、思维导图和 study artifact 分支需要 NotebookLM 或兼容 CLI。微信草稿、小红书文稿和内容诊断本身不依赖 NotebookLM。

## 必须使用 md2wechat 吗？

微信草稿上传分支需要 `md2wechat` 或等价的微信公众平台草稿上传工具。没有该工具时，仍可生成改写稿、HTML 和封面，但不能自动创建草稿。

## 生成播客会自动创建微信草稿吗？

不会。播客分支和微信草稿分支是两个独立层面。只有明确要求“生成播客，并上传微信公众号草稿”时才会同时运行。

## 为什么现在会先生成内容诊断？

因为同一篇源内容在播客、公众号、小红书和 PPT 里的表达方式不同。诊断文件会先判断内容价值、传播价值、认知落差、平台策略和风险点，再让后续分支按同一个判断生成，避免每个分支各写各的。

## 微信草稿里会自动嵌入音频吗？

默认不会。当前稳定流程会生成本地播客音频，并创建微信文章草稿。音频嵌入需要 `md2wechat capabilities --json` 明确支持音频或 voice 素材上传与插入。

## 为什么要先分析再改写？

公众号文章、小红书科普帖、播客和 PPT 都不是简单格式转换。直接转换或轻微改写容易保留原文结构，不适合作为新的平台内容。

## PPT 或思维导图会只交付 Markdown 吗？

不应只交付 Markdown。Markdown 可以作为中间稿，但完成品应是 `.pptx`、`.png`、`.svg`，或可验证的 NotebookLM native artifact。

## 小红书会自动发布吗？

不会。当前小红书分支默认生成本地 Markdown/JSON，包括标题候选、正文、标签和封面建议。除非未来接入明确的小红书发布工具，否则不自动上传。

## 播客为什么默认是 `.m4a`？

当前稳定的 NotebookLM 下载链路通常返回 `.m4a`。不要把 `.m4a` 直接改名成 `.mp3`；只有真实转码后才应交付 MP3。

## 可以处理付费墙内容吗？

项目不会绕过权限控制。你可以处理自己有权限访问、已导出的正文、笔记或转录稿。对于无法公开访问的链接，请先合法导出为 Markdown/TXT。

## 微信接口返回 invalid ip 怎么办？

把错误信息里的 IP 加入微信公众平台后台的 IP 白名单，然后重试上传草稿。

## 是否可以提交真实配置文件？

不可以。`md2wechat.yaml`、`.env`、Cookie、token、AppID/Secret 等都应保存在本地，并已被 `.gitignore` 排除。
